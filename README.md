# 企業午餐訂餐系統 V7 Lock2 測試版

## 測試帳號

- C5454 / KOOK：一般員工
- B001 / 林主管：主管
- C001 / 陳助理：助理

## 本版重點

- 頁面一開啟會先判斷是否超過 10:00。
- 若已超過截止時間，會直接顯示「目前未開放訂餐」。
- 不會先進入工號 / 姓名查詢畫面。
- 超過截止時間後，員工端無法掃 QRCode、驗證、建立或修改訂單。
- 若使用中超過 10:00，會自動切換到鎖定畫面。
- 測試時如需暫時解除鎖定，可在 app.js 將 TEST_FORCE_OPEN 改為 true。
